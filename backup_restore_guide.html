Section 1 — Understanding the System

Your book store has 3 types of data that must be backed up together. If even one is missing, the restore will be incomplete.

1. Database (Book records, Orders, Customers)

This is the "brain" of the store. It stores every book's title, price, author, and the exact filename of its cover image. If this is lost, you lose all product info and all order history.

LocationPath / Location
Local computerc:\Project_AI\iskon_book_store\iskcon_books.db
Railway (live site)Managed by Railway — export from Railway dashboard → PostgreSQL → Backups

2. Book Cover Images

These are the .jpg / .png image files for each book. They are saved with random code names like a1b2c3d4e5f6.jpg. Do NOT rename them — the database uses these exact names to find the right image for each book.

LocationPath / Location
Local computerc:\Project_AI\iskon_book_store\static\images\books\
Railway (live site)Stored on Railway's persistent volume (survives restarts)

3. eBook Files (PDF / EPUB) — if any uploaded

Same principle as images — stored with random filenames, must not be renamed.

LocationPath / Location

Railway (live site)Railway's persistent volume

Section 2 — How Images Link to Books

When you upload a book cover image in the admin panel, the system automatically:

  1. Renames the file to a unique random code like a1b2c3d4.jpg
  2. Saves the file in the static/images/books/ folder
  3. Records that random filename in the database against that specific book

This means "Bhagavad Gita" in the database has image = a1b2c3d4.jpg. As long as the file a1b2c3d4.jpg is in the images folder, the correct cover shows on the website — automatically, for the right book.

⚠️ Critical Rule: Never rename image files in the books folder. The random code name IS the permanent link between the image file and its book. Renaming breaks that link.

Section 3 — Taking a Backup (Local Computer)

Do this backup once a week or after making major changes (adding many books, bulk image uploads, etc.).

1 Backup the Images Folder

  1. Open File Explorer
  2. Go to: c:\Project_AI\iskon_book_store\static\images\books\
  3. Right-click the books folder → Send to → Compressed (zipped) folder
  4. Name it: book_images_backup_2026-04-19.zip (use today's date)
  5. Move this zip to a safe place: USB drive, Google Drive, or an external hard disk

2 Backup the eBooks Folder (only if you have uploaded PDFs)

  1. Go to: c:\Project_AI\iskon_book_store\ebooks\
  2. Right-click → Send to → Compressed (zipped) folder
  3. Name it: ebooks_backup_2026-04-19.zip
  4. Move to the same safe location

3 Backup the Database File

  1. Go to: c:\Project_AI\iskon_book_store\
  2. Find the file: iskcon_books.db
  3. Copy it and rename the copy: iskcon_books_backup_2026-04-19.db
  4. Move to the same safe location
✅ Verify your backup folder contains these 3 files:
  • book_images_backup_2026-04-19.zip
  • ebooks_backup_2026-04-19.zip (only if eBooks were uploaded)
  • iskcon_books_backup_2026-04-19.db

Section 4 — Restoring (Local Computer)

Use this when something goes wrong on your local computer — such as accidental deletion, corruption, or a fresh Windows install.

1 Restore the Database

  1. Take your backup file: iskcon_books_backup_2026-04-19.db
  2. Rename it to: iskcon_books.db
  3. Paste it into: c:\Project_AI\iskon_book_store\
  4. Click Yes if asked to replace the existing file

2 Restore the Images

  1. Open your backup zip: book_images_backup_2026-04-19.zip
  2. Extract all files from it
  3. Copy all extracted .jpg / .png files into:
    c:\Project_AI\iskon_book_store\static\images\books\
  4. Do NOT replace default_book.jpg if it is already there

3 Restore eBooks (only if applicable)

  1. Open your backup zip: ebooks_backup_2026-04-19.zip
  2. Extract all files into: c:\Project_AI\iskon_book_store\ebooks\

4 Restart the Application

  1. Open a terminal / command prompt in the project folder
  2. Run: python app.py
  3. Open http://127.0.0.1:5000 in your browser
  4. All images will show correctly for the right books — automatically

Restore Images to Railway

If images ever disappear from the live site (Railway volume reset or similar issue), follow these steps:

Steps to push all images back to Railway:
  1. Make sure your local images are restored in:
    c:\Project_AI\iskon_book_store\static\images\books\
  2. Open a terminal in the project folder
  3. Run: python upload_images_to_railway.py
  4. Wait for it to finish — it uploads every image to the Railway server one by one
  5. Refresh iskconbooks.in — all covers will appear correctly
ℹ️ What the upload script does: It reads every image file from your local static/images/books/ folder and sends each file to the live Railway server, preserving the exact filenames that the database expects. Each book's image links up automatically.

Section 8 — Important Rules to Remember

  • 🚫 Never rename image files in the static/images/books/ folder — the random code name is the permanent link between image and book
  • 📅 Always backup the DB and images together on the same date — they must match each other
  • 💾 Keep at least 3 recent backups — if the latest is corrupted, you have older ones to fall back to
  • 📸 After any bulk image upload, take a fresh backup immediately
  • ☁️ The upload script (upload_images_to_railway.py) only pushes images to Railway — it does not change the database
  • 🗄️ DB and images must match — if you restore an old DB, you may also need matching old images from the same date
  • 🔑 Keep your .env file safe — it contains your admin password and payment keys; back it up separately
